Overview

The core of this project is a C++ sentence parser. It reads an input sentence, normalizes it, looks up the words in a lexicon, applies a grammar, and builds one or more parse trees. If a sentence is ambiguous, the parser keeps multiple valid interpretations instead of forcing one answer (which is the cool part!)

Project repository: github.com/yzdu0/sentence-reader

Algorithm used

Mathematically, the program uses a Context-Free Grammar to parse a sentence. In the Reference section of this website, there is a list of grammar rules such as S -> NP VP. S denotes entire sentence, NP denotes "Noun Phrase" - a chunk of our sentence that functions as a noun (person place or thing) and VP denotes "Verb Phrase" - a chunk of our sentence that functions as a verb (an action)

For the sentence "The cat saw the man", the first phrase "The cat" is our noun phrase and "saw the man" is the verb phrase. Syntactically, combining a noun phrase and a verb phrase makes a sentence that sounds right - hence the rule S -> NP VP is present in our program.

The most natural way for our program to find this breakdown is by recursively applying every single possible rule. We begin with the assumption that this is a functional sentence, that is, "The cat saw the man" can be explained by rule "S -> NP VP"

Now imagine a divider between two words that dictates where the NP ends and the VP ends. There are a few options, but the program will try all possibilities:
" | The cat saw the man", "The | cat saw the man", "The cat | saw the man"...

We have six trial explanations in total now. (four if we disregard the cases where one phrase is empty.)

Now, let's consider the third sentence "The cat | saw the man" as we know this is the correct one. The program recursively does the same process to both parts of the sentence; it tries to explain "The cat" as a Noun Phrase, and explain "saw the man" as a Verb Phrase. (The program also does this to the other five trial explanations, as it doesn't yet know which is correct, of course).

For the Noun Phrase, we have a few rules:
NP -> Pron
NP -> Det N
NP -> Det AdjP N
"Det" refers to determiner; the word "the" in this phrase.
The program will then loop through all rules with NP on the left hand side. For every single rule, it will recursively try every single segmentation. It will find that the second one (NP -> Det N) works with the segmentation "The | cat". As we are trying to explain "The" as a Det and "cat" as a N (Noun), we have reached the two base cases; as we are now only trying to explain single words.

C++ implementation

The time complexity of the above is pretty ridiculous but dynamic programming (caching results) allows this to be done in polynomial time. Additionally, I ended up switching to a bottom-up method rather than recursion, as it already has nice research done on it, and was easier to extend this to find ALL interpretations.
